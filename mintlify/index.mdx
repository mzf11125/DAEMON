## What you get

| Capability | Where it lives |
|------------|----------------|
| Ingest and normalize | `collect-sensing` (Go) |
| Ontology registry, graph, vectors | `ontology` (Go + Rust) |
| Reads, writes, approvals | `read-write-loops` (TypeScript) |
| Workflows and agents | `action-runtime` (TypeScript) |
| HTTP API | NestJS `api/gateway` |
| Policy and audit | `security-governance` |

## Quality bar

- TypeScript, Go, and Rust implementations — no stub handlers in integration paths.
- Integration and e2e tests use **testcontainers** / docker-compose — not mocked upstreams.
- Publishable packages: `@daemon/platform-types`, `@daemon/sdk`, `@daemon/cli`.
